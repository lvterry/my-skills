# Blog Publisher

Publish a new Agent Economy article from a URL.

## Current repo structure

- Blog repo: `/root/.openclaw/workspace/agent-economy-blog`
- Articles now use Astro Content Collections, not one `.astro` page per post.
- Create exactly two Markdown files for each new article, with the same slug:
  - Chinese: `src/content/posts/zh/<slug>.md`
  - English: `src/content/posts/en/<slug>.md`
- Do **not** create `src/pages/blog/<slug>.astro`.
- Do **not** edit `src/data/posts.ts`; that file was removed.
- Do **not** manually edit RSS files for ordinary posts. RSS is generated from `getCollection('posts')` in:
  - `src/pages/rss.xml.js`
  - `src/pages/en/rss.xml.js`
- Dynamic article routes are generated by:
  - `src/pages/blog/[slug].astro` for Chinese `/blog/<slug>/`
  - `src/pages/en/blog/[slug].astro` for English `/en/blog/<slug>/`
- Shared article rendering lives in `src/components/pages/PostPage.astro`.
- Category mapping lives in `src/lib/categories.ts`; update it only if a new tag needs to appear under a category filter.

## Default workflow

1. **Start clean and up to date.**
   - In the blog repo, run `git status --short --branch`.
   - Pull latest code with `git pull --ff-only` if safe.
   - If local generated files block the pull, stash them with a descriptive message instead of discarding them.

2. **Fetch and understand the source.**
   - Read the source article/page fully before writing.
   - If browser navigation times out or dynamic docs pages are hard to inspect, use `requests` + BeautifulSoup from terminal to extract the static HTML article content.
   - For Chinese Docusaurus pages, explicitly set `r.encoding = 'utf-8'` after `requests.get(...)`; otherwise extracted Chinese text can become mojibake even when the page is valid UTF-8.
   - If extraction is incomplete, use another source, localized URL, or browser inspection before drafting.
   - Do not rely on old helper scripts (`scripts/fetch_article.py`, `scripts/generate_content.py`, `scripts/add_article.py`); the scripts directory may not exist in the new structure.

3. **Choose slug, date, source, and tags.**
   - Slug: lowercase kebab-case, stable, descriptive, shared by zh/en files.
   - Date: `YYYY-MM-DD` string.
   - Source: short publisher/company name, e.g. `OpenAI`, `Anthropic`, `Google`.
   - Tags: use an array of strings. Prefer existing category tags from `src/lib/categories.ts` when possible so category filters work.

4. **Write final bilingual copy yourself.**
   - Chinese and English are separate complete Markdown articles, not mixed in one file.
   - Each file needs frontmatter matching `src/content/config.ts`:

```md
---
title: 'Article title'
excerpt: 'One or two sentence rewritten excerpt.'
date: "YYYY-MM-DD"
tags: ["Tag1", "Tag2", "Tag3"]
source: "Source Name"
---

Article body paragraphs in this language.

[阅读原文](https://original-url)
```

English file ends with:

```md
[Read the full article](https://original-url)
```

5. **Create paired content files.**
   - `src/content/posts/zh/<slug>.md`
   - `src/content/posts/en/<slug>.md`
   - The frontmatter fields should be aligned across languages except `title` and `excerpt`.
   - Keep the same `date`, `tags`, and `source` in both files.

6. **Build before publishing.**
   - Run `npm run build` in `/root/.openclaw/workspace/agent-economy-blog`.
   - The build should generate both routes:
     - `/blog/<slug>/`
     - `/en/blog/<slug>/`
   - If build fails, fix errors and rerun until it passes.

7. **Commit only intended source changes and push.**
   - Usually commit only the two new Markdown files, plus `src/lib/categories.ts` if category mapping was intentionally updated.
   - Do not commit unrelated `dist/`, `test-results/`, node_modules, or stashed/generated files.
   - Run `git diff --stat` and `git status --short` before commit.

## Editorial rules

### Titles

- Use factual, descriptive titles.
- Avoid clickbait patterns: ALL CAPS hype words, exaggerated superlatives, or vague drama.
- Preferred shape: Company/Subject + Action + Key Detail.
- Do **not** use colons in final Chinese or English titles.
- Keep titles natural, specific, and readable.
- Avoid mechanical translation of the source headline.

### Excerpts

- Write fresh Chinese and English excerpts.
- Keep them to 1-2 sentences.
- Summarize both the fact and why it matters.
- Do not copy the source deck verbatim.

### Article body

- Write from an informed editorial perspective, not as a raw summary.
- Add context, implications, or strategy when useful.
- Prefer short paragraphs.
- Usually avoid bullet lists unless the source is highly technical and a short list genuinely improves clarity.
- Keep the piece concise and worth reading for a busy AI-native audience.

### Voice

- Match Agent Economy themes: AI infrastructure, security, trust, commercialization, applications, and trading.
- Optimize for signal over fluff.
- The article should feel curated, not auto-generated.

## Quality checklist before pushing

- Exactly two new post files exist under `src/content/posts/zh/` and `src/content/posts/en/` with the same slug.
- Frontmatter has `title`, `excerpt`, `date`, `tags`, and `source`.
- Titles contain no colon and avoid hype/clickbait.
- Excerpts are rewritten, not copied.
- Chinese and English articles each include the original source link.
- Tags are sensible and map to categories where possible.
- `npm run build` passes.
- Build output includes both zh and en routes for the slug.
- Commit contains only intended source changes.
